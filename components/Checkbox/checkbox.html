Anatomy

Komponen = control box (16×16) + label (.ds-checkbox__label, body small medium) dengan gap: space-100. Ikon check/minus hanya pada status checked / indeterminate.

A Control box
Unchecked
Ukuran 16×16px, radius border-radius-sm. Default: n0 + border n40. Hover: fill n20 + border n0.
A Control box
B Icon slot
Checked / Indeterminate
Fill Background.Brand.Bold (b300). Ikon icon-check.svg atau icon-minus.svg 12×12px, Icon.Inverse via filter. Error: ring Border.Danger (r300) 2px di luar kotak.

Informasi Penggunaan

Kapan dipakai
Untuk memilih satu atau beberapa opsi independen dalam form, tabel bulk-select, atau filter. Gunakan radio jika hanya satu pilihan dalam grup.
Parts & komponen
Parts Figma: appearance=default/hover, isIndeterminate, isChecked, invalid. Komponen: matrix state (default → hover → pressed → disable → focus) × status boolean Figma.
Aksesibilitas
Selalu gunakan <input type="checkbox"> native yang disembunyikan secara visual, dengan <label> atau aria-labelledby. Jangan mengandalkan warna saja untuk state error — sertakan pesan teks.

Do & Don't

Do — Lakukan ini
  • Gunakan ukuran kotak 16×16px dan radius border-radius-sm sesuai token Corner untuk form control.
  • Pakai icon-check.svg dan icon-minus.svg dari assets/icons untuk checked dan indeterminate.
  • Tampilkan ring error (Border.Danger) hanya saat validasi gagal, bukan sebagai gaya dekoratif.
  • Hover unchecked memakai fill n20; pressed unchecked memakai n40; focus memakai ring b200 (bukan fill gelap).
  • Hubungkan label teks ke input agar area klik mencakup teks (gap space-100).
Don't — Hindari ini
  • Jangan membuat kotak checkbox hanya dari div tanpa input native — aksesibilitas keyboard dan screen reader akan rusak.
  • Jangan hardcode warna hex — gunakan class token atau alias Tailwind DS (bg-b300, border-n40).
  • Jangan gunakan indeterminate untuk state “checked” penuh — itu hanya untuk seleksi parsial.
  • Jangan mengecilkan target sentuh di mobile; pertahankan label yang cukup besar di sekitar kotak 16px.
  • Jangan menampilkan error ring pada checkbox yang belum disentuh pengguna (hindari error prematur).

Code — Salin Code

checkbox.html 
<!-- Part: checked (static) -->
<span class="ds-checkbox__box ds-checkbox__box--checked" aria-hidden="true">
  <img src="../../assets/icons/icon-check.svg" alt="" class="ds-checkbox__icon ds-checkbox__icon--check" aria-hidden="true">
</span>

<!-- Komponen: label + input -->
<label class="ds-checkbox">
  <input type="checkbox" class="ds-checkbox__input">
  <span class="ds-checkbox__box" aria-hidden="true">
    <img src="../../assets/icons/icon-check.svg" alt="" class="ds-checkbox__icon ds-checkbox__icon--check">
    <img src="../../assets/icons/icon-minus.svg" alt="" class="ds-checkbox__icon ds-checkbox__icon--minus">
  </span>
  <span class="ds-checkbox__label">Label</span>
</label>

<!-- Error unchecked -->
<label class="ds-checkbox ds-checkbox--error">
  <input type="checkbox" class="ds-checkbox__input" aria-invalid="true">
  ...
</label>

<!-- Error checked -->
<label class="ds-checkbox ds-checkbox--error">
  <input type="checkbox" class="ds-checkbox__input" checked aria-invalid="true">
  ...
</label>
