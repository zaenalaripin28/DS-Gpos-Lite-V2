Informasi Penggunaan

Kapan dipakai
Untuk status semantik pada baris tabel, issue tracker, workflow, atau metadata item — bukan untuk angka hitung (gunakan Badge).
Subtle vs Bold
Subtle untuk daftar padat dan kontras rendah. Bold saat status perlu menonjol di header, filter aktif, atau highlight tunggal.
Appearance
Enam appearance memetakan makna: default, inprogress, moved, new, removed, success. Jangan ubah makna warna antar konteks.

Do & Don't

Do — Lakukan ini
  • Gunakan teks singkat UPPERCASE (1–2 kata), mis. OPEN, DONE.
  • Pilih appearance sesuai status bisnis yang sudah didefinisikan di Figma.
  • Gunakan subtle di tabel/list; bold hanya untuk penekanan.
  • Pertahankan padding space-025 / space-100 dan radius border-radius-sm.
Don't — Hindari ini
  • Jangan isi lozenge dengan kalimat panjang atau paragraf.
  • Jangan hardcode warna hex — pakai class token / Tailwind alias DS.
  • Jangan pakai bold untuk semua sel tabel (mengurangi hierarki).
  • Jangan gunakan lozenge sebagai tombol aksi — gunakan Button.

Tailwind — Salin Code

Class dasar + modifier per appearance. Warna mengacu alias di tailwind.config halaman ini.

Blok siap copy (Tailwind)

lozenge-tailwind.html 
<!-- Base + appearance subtle -->
<span class="inline-flex items-center rounded px-2 py-0.5 text-xs font-semibold uppercase tracking-wider bg-b50 text-b300">In Progress</span>

<!-- Bold modifier -->
<span class="inline-flex items-center rounded px-2 py-0.5 text-xs font-semibold uppercase tracking-wider bg-b300 text-n0">In Progress</span>

<!-- Semua appearance (subtle) -->
<span class="inline-flex items-center rounded px-2 py-0.5 text-xs font-semibold uppercase tracking-wider bg-n20 text-n500">Default</span>
