Informasi Anatomy
  • A. Trigger Field: area interaksi utama untuk membuka/menutup list option.
  • B. Value Slot: menampilkan placeholder, single value, atau multi-value (tag).
  • C. Chevron Icon: indikator status dropdown (collapsed/expanded).
  • D. Dropdown Panel: kontainer option dengan state hover/selected/loading/empty.
Informasi Penggunaan
  • Gunakan Select untuk pilihan terstruktur dengan jumlah item sedang hingga banyak.
  • Gunakan single-select jika hanya boleh satu nilai; gunakan multi-select saat butuh kombinasi nilai.
  • Tampilkan state loading/empty secara eksplisit agar pengguna memahami status data option.
Do (Lakukan)
  • Gunakan state focus, invalid, disabled sesuai kondisi form yang nyata.
  • Pertahankan struktur trigger + dropdown + option secara konsisten di semua varian.
  • Pastikan label value singkat agar tidak memotong affordance ikon dan tag.
Don't (Hindari)
  • Jangan hardcode warna border/state di luar token semantik yang tersedia.
  • Jangan campur pola single-select dan multi-select dalam satu trigger yang sama.
  • Jangan sembunyikan feedback error; state invalid harus tetap terlihat jelas.

Informasi Relevan — Option Part
  • Option Part adalah fondasi seluruh varian Select (default, checkbox, radio, icon) dengan state konsisten: default, hover, pressed, selected, selected-hover.
  • Gunakan struktur base yang sama untuk semua turunan agar transisi antar varian tetap seragam dan mudah dipelihara.
  • Untuk kebutuhan list panjang, gunakan turunan Select (Scrollable) atau Group (Scrollable) tanpa mengubah token inti Option Part.

Anatomy

Informasi Anatomy
  • A. Row Container: area option dengan tinggi minimum 40px dan padding horizontal tokenized.
  • B. Label: text body-small sebagai konten utama.
  • C. Control Slot (opsional): checkbox/radio indicator untuk multi/single selection.
  • D. Icon Slot (opsional): trailing 1 atau 2 icon sesuai kebutuhan konteks option.

Micro Interaction

  • Hover: background berpindah ke neutral surface yang sedikit lebih kontras.
  • Pressed: label pindah ke warna brand untuk memberi feedback sentuhan/klik.
  • Selected: background ke white dan teks brand sebagai state terpilih yang stabil.
  • Selected Hover: mempertahankan selected sambil menambahkan tint biru untuk affordance interaktif.

Informasi Penggunaan

Section Information

Option part dipakai sebagai building block list pada menu Select. Struktur ini konsisten untuk default item, item dengan checkbox/radio, dan item dengan icon actions.

Aksesibilitas

Pastikan state selected tetap terbaca jelas, area klik cukup besar, dan label tetap ringkas agar scanning list tetap cepat.

Do & Don't

Do (Lakukan)
  • Gunakan state visual sesuai urutan interaksi: default → hover → pressed → selected.
  • Pakai checkbox/radio hanya saat tipe pemilihan memang membutuhkannya.
  • Pertahankan jarak dan ukuran option sesuai token spacing yang sudah ada.
  • Gunakan icon slot sebagai pelengkap, bukan pengganti label.
Don't (Hindari)
  • Jangan hardcode warna di luar token semantic yang sudah tersedia.
  • Jangan ubah pattern selected menjadi style baru yang berbeda dengan komponen input lain.
  • Jangan menaruh terlalu banyak icon hingga label sulit dibaca.
  • Jangan menurunkan tinggi option sehingga target klik menjadi sempit.
