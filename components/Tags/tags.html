Informasi Penggunaan

Kapan dipakai
Untuk menandai kategori, filter aktif, metadata singkat, atau daftar pilihan yang bisa dihapus pengguna — bukan untuk status workflow (gunakan Lozenge) atau angka (gunakan Badge).
Varian standard
Default semua varian: Background.Neutral. Hover netral: Background.Neutral.Pressed; pressed: Background.Neutral.Bold.Pressed. Removable hover/pressed: Background.Danger.Tag / .Hovered + Text.Accent.Red; focus removable: kembali Background.Neutral.
Varian bluelight
Default/focus: Background.Accent.Blue.Subtler (b75). Hover netral: Background.Accent.Blue.Subtlest (b50). Removable hover/pressed: Background.Danger.Tag.Hovered + Text.Accent.Red; focus removable: Background.Accent.Blue.Subtler.
Varian redlight
Default/focus: Background.Accent.Red.Subtler (r75). Hover: Background.Accent.Red.Subtlest (r50). Pressed: Background.Accent.Red.Subtle (r100) + Text.Accent.Red. Removable hover/pressed: Background.Danger.Tag.Hovered (r75) + Text.Accent.Red.
Konten & ikon
Empat struktur konten: teks saja, ikon + teks, teks + remove, ikon + teks + remove. Class: ds-tag--standard, ds-tag--bluelight, atau ds-tag--redlight + ds-tag--removable.

Do & Don't

Do — Lakukan ini
  • Gunakan teks pendek (1–3 kata), mis. Elektronik atau Jakarta.
  • Pakai ds-tag--removable hanya jika pengguna benar-benar bisa menghapus tag.
  • Berikan aria-label pada tombol remove, mis. Hapus tag Elektronik.
  • Pertahankan padding space-050 / space-100 dan radius border-radius-full.
Don't — Hindari ini
  • Jangan isi tag dengan kalimat panjang atau paragraf.
  • Jangan hardcode warna hex — pakai token / alias Tailwind DS.
  • Jangan tampilkan ikon remove tanpa fungsi hapus yang jelas.
  • Jangan hilangkan focus-visible pada tag interaktif dan tombol remove.
