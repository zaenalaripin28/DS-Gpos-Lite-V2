Informasi Penggunaan

Kapan dipakai
Untuk menandai kategori, filter aktif, metadata singkat, atau daftar pilihan yang bisa dihapus pengguna — bukan untuk status workflow (gunakan Lozenge) atau angka (gunakan Badge).
Varian standard
Default semua varian: Background.Neutral. Hover netral: Background.Neutral.Pressed; pressed: Background.Neutral.Bold.Pressed. Removable hover: Background.Danger.Tag + Text.Danger; pressed: Background.Danger.Tag.Hovered + Text.Accent.Red; focus: kembali Background.Neutral.
Varian bluelight
Default/focus netral: Background.Accent.Blue.Subtler (b75). Hover netral: Background.Accent.Blue.Subtlest (b50). Removable: token sama standard (hover Danger.Tag + Text.Danger; focus Background.Neutral).
Varian redlight
Latar: Accent.Red.Subtler / Subtlest (hover) / Subtle (pressed). Teks & ikon: Text.Accent.Red (r500) — kecuali hover pakai Text.Danger (r300). Removable bg sama standard.
Varian greenlight
Default/focus: Background.Accent.Green.Subtler + Text.Accent.Green. Hover netral: Background.Accent.Green.Subtlest + Text.Success. Removable hover: sama standard (Danger.Tag + Text.Danger).
Varian greylight
Default/focus: Background.Accent.Gray.Subtler (n40) + Text.Bold. Hover netral: Background.Accent.Gray.Subtlest (n20) + Text.Subtle. Pressed netral: Background.Accent.Gray.Subtle (n70). Removable hover/pressed: global Danger.Tag (sama greenlight).
Varian yellowlight
Default/focus: Background.Accent.Yellow.Subtler + Text.Accent.Yellow. Hover netral: Background.Accent.Yellow.Subtlest + Text.Warning. Pressed: Background.Accent.Yellow.Subtle. Removable hover/pressed: sama greenlight (Danger.Tag global).
Varian purplelight
Default/focus: Background.Accent.Purple.Subtler + Text.Accent.Purple. Hover netral: Background.Accent.Purple.Subtlest + Text.Discovery. Pressed netral: Background.Accent.Purple.Subtle. Removable hover/pressed: sama greenlight (Danger.Tag global).
Konten & ikon
Empat struktur konten: teks saja, ikon + teks, teks + remove, ikon + teks + remove. Class: ds-tag--standard, ds-tag--bluelight, ds-tag--redlight, ds-tag--greenlight, ds-tag--greylight, ds-tag--yellowlight, ds-tag--purplelight + ds-tag--removable.

Do & Don't

Do — Lakukan ini
  • Gunakan teks pendek (1–3 kata), mis. Elektronik atau Jakarta.
  • Pakai ds-tag--removable hanya jika pengguna benar-benar bisa menghapus tag.
  • Berikan aria-label pada tombol remove, mis. Hapus tag Elektronik.
  • Pertahankan padding space-050 / space-100 dan radius border-radius-full.
Don't — Hindari ini
  • Jangan isi tag dengan kalimat panjang atau paragraf.
  • Jangan hardcode warna hex — pakai token / alias Tailwind DS.
  • Jangan tampilkan ikon remove tanpa fungsi hapus yang jelas.
  • Jangan hilangkan focus-visible pada tag interaktif dan tombol remove.
