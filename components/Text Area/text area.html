Text Area

Komponen input multi-baris untuk konten panjang seperti catatan, deskripsi, dan alasan. Mengikuti semantic color token, spacing token, typography token, dan interaction pattern yang sama seperti input komponen lain.

Multi-line input Hover / Focus / Invalid / Disabled Figma-synced

Varian & State

Gunakan state sesuai konteks interaksi komponen.

Default
Focus
Invalid
Disabled

Anatomy

Informasi Anatomy
  • A. Container: wrapper komponen dengan lebar mengikuti parent.
  • B. Textarea Field: area input multi-baris dengan min-height dan vertical resize.
  • C. Placeholder / Value: menggunakan typography token body-small.
  • D. Border & Focus Ring: border input dan ring fokus mengikuti interaction pattern text input.

Micro Interaction

  • Hover: border berubah ke var(--color-neutral-n100).
  • Focus: border berubah ke var(--color-blue-b300) dan ring 2px var(--color-blue-b200).
  • Invalid focus: ring menggunakan token danger untuk menjaga feedback error tetap jelas.
  • Disabled: background dan text pindah ke token neutral-disabled serta cursor not-allowed.

Informasi Penggunaan

Section Information

Text Area digunakan saat pengguna perlu menulis input panjang (lebih dari satu baris), misalnya catatan tambahan, alasan, komentar, atau deskripsi produk.

Aksesibilitas

Gunakan label yang jelas, placeholder sebagai bantuan singkat, dan state invalid dengan pesan error terpisah agar keterbacaan serta validasi tetap konsisten.

Do & Don't

Do (Lakukan)
  • Gunakan Text Area untuk input panjang/multi-baris.
  • Pertahankan tinggi minimal agar konten awal tetap terbaca.
  • Aktifkan state invalid saat validasi gagal.
  • Ikuti token spacing dan typography yang sudah ada.
Don't (Hindari)
  • Jangan pakai Text Area untuk input satu kata/sangat pendek.
  • Jangan hardcode warna border, text, atau background.
  • Jangan hilangkan fokus ring karena mengurangi aksesibilitas.
  • Jangan ubah pattern interaksi berbeda dari komponen input lain.
