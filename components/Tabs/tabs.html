Informasi Penggunaan

Panduan singkat memasang Tabs di halaman produk GPOS Lite.

Kapan dipakai
Gunakan saat konten dibagi 2–6 bagian setara (mis. Ringkasan, Detail, Riwayat). Jangan ganti navigasi utama aplikasi — itu peran menu/sidebar.
Struktur HTML
role="tablist" pada wrapper · role="tab" + aria-selected pada tiap tombol · panel konten terkait memakai role="tabpanel" dan aria-labelledby.
Part item tab
default (tidak aktif) · selected (aktif) · required (asterisk) · notification (badge). Kombinasikan dengan layout track sesuai kebutuhan visual halaman.
Layout track
Pilih ds-tablist--track bila tab berada di atas konten kartu/form dan perlu pemisah horizontal yang jelas. Tanpa track untuk toolbar atau sub-nav yang lebih ringan.
Interaksi
Klik mengubah tab aktif. Keyboard: panah kiri/kanan pindah fokus, Enter/Space memilih. Saat pindah tab, update aria-selected dan class ds-tab--selected.
Notifikasi & wajib
Badge notifikasi memakai ds-badge--important dari komponen Badge. Asterisk wajib hanya untuk tab yang kontennya belum lengkap atau wajib diisi pengguna.

Do & Don't

Panduan penggunaan Tabs yang benar agar navigasi antar-panel tetap jelas dan konsisten dengan design system.

Do — Lakukan ini
Hanya satu tab aktif
Selalu tepat satu aria-selected="true" per tablist. Indikator biru hanya pada tab aktif.
Label singkat & bermakna
1–2 kata yang menjelaskan panel di bawahnya, bukan kalimat panjang atau instruksi.
Badge untuk hitungan belum dibaca
Gunakan ds-badge--important hanya bila ada angka/notifikasi nyata. Hilangkan badge setelah pengguna membuka panel.
Track di atas area konten
Pasang ds-tablist--track saat tab memisahkan blok konten besar (form, tabel, kartu).
Don't — Hindari ini
Jangan dua tab aktif sekaligus
Lebih dari satu indikator biru membingungkan pengguna tentang panel mana yang sedang ditampilkan.
Jangan label terlalu panjang
Teks panjang merusak ritme horizontal tab. Ringkas atau pindah ke tooltip/breadcrumb.
Jangan lebih dari ~6 tab
Terlalu banyak pilihan sejajar sulit discan. Pertimbangkan dropdown atau sub-nav untuk kategori tambahan.
Jangan asterisk tanpa alasan
ds-tab--required hanya jika tab memang wajib diisi/diselesaikan. Jangan untuk sekadar menarik perhatian.

Kode HTML (ds-tab)

Copy HTML per part. Bungkus dalam .ds-tablist atau .ds-tablist.ds-tablist--track.

tabs.html 
<!-- Tablist + track (disarankan di atas konten) -->
<div class="ds-tablist ds-tablist--track" role="tablist" aria-label="Sections">
  <button type="button" class="ds-tab" role="tab" aria-selected="false">...</button>
  <button type="button" class="ds-tab ds-tab--selected" role="tab" aria-selected="true">...</button>
</div>

<!-- part=default -->
<button type="button" class="ds-tab" role="tab" aria-selected="false">
  <span class="ds-tab__label">Tab</span>
  <span class="ds-tab__indicator" aria-hidden="true"></span>
</button>

<!-- part=selected -->
<button type="button" class="ds-tab ds-tab--selected" role="tab" aria-selected="true">
  <span class="ds-tab__label">Tab</span>
  <span class="ds-tab__indicator" aria-hidden="true"></span>
</button>

<!-- part=default:required -->
<button type="button" class="ds-tab ds-tab--required" role="tab" aria-selected="false">
  <span class="ds-tab__row">
    <span class="ds-tab__label">Tab</span>
    <span class="ds-tab__required" aria-hidden="true">*</span>
  </span>
  <span class="ds-tab__indicator" aria-hidden="true"></span>
</button>

<!-- part=default:notification (badge dari Badge component) -->
<button type="button" class="ds-tab ds-tab--notification" role="tab" aria-selected="false">
  <span class="ds-tab__row">
    <span class="ds-tab__label">Tab</span>
    <span class="ds-tab__badge"><span class="ds-badge ds-badge--important">1</span></span>
  </span>
  <span class="ds-tab__indicator" aria-hidden="true"></span>
</button>
