Documentation & Usage

Production-ready Component

Copy-ready button using .ds-btn with all interactive states wired via tokens.css custom properties.

CSS — .ds-btn using tokens.css
/* tokens from tokens.css */
.ds-btn {
  background-color: var(--color-neutral-n20);
  color:            var(--color-neutral-n900); /* --Color-Text-Bold */
  font-family:      var(--font-primary);       /* Poppins */
  font-size:        var(--font-size-sm);       /* 14px */
  font-weight:      var(--font-weight-medium); /* 500 */
  line-height:      var(--text-body-small-medium-lh); /* 20px */
  word-wrap:        break-word;
  padding:          0 var(--space-150);        /* 12px */
  min-height:       32px;
  border-radius:    6px;
  transition:       background-color var(--transition-fast);
}
.ds-btn:hover   { background-color: var(--color-neutral-n30); }
.ds-btn:active  { background-color: var(--color-neutral-n40); }
.ds-btn:disabled { color: var(--color-neutral-n50); }
HTML
<button type="button" class="ds-btn">Button</button>

<!-- Selected -->
<button type="button" class="ds-btn" aria-pressed="true">Selected</button>

<!-- Disabled -->
<button type="button" class="ds-btn" disabled>Disabled</button>

Usage Guidance

Do
Use disabled attribute on the <button> element so screen readers announce the disabled state natively.
Don't
Never use <div> or <a> for action triggers — use a semantic <button>.
Do
Add aria-pressed="true" when isSelected=true to communicate toggle state to assistive technology.
Don't
Don't suppress focus with outline:none without providing a custom focus indicator — .ds-btn:focus-visible handles this correctly.
