Kapan dipakai
Form registrasi, profil pelanggan, verifikasi OTP, atau checkout yang membutuhkan nomor HP Indonesia (+62). Jangan dipakai jika pengguna perlu memilih kode negara lain — pertimbangkan combobox terpisah.
Format input
Placeholder menampilkan contoh nomor lokal (81234567890) tanpa prefix. Validasi di backend gabungkan +62 + input pengguna. Terima hanya digit; format tampilan opsional.
Do & Don't singkat
Do: label "Nomor telepon", autocomplete="tel-national", pesan error di bawah.
Don't: ulangi +62 di placeholder, atau biarkan prefix bisa dihapus pengguna.

Text field terdiri dari container, input area, border/underline, dan focus ring opsional. Pasangkan label terpisah di atas field — bukan placeholder sebagai satu-satunya label.

Standard & subtle: border 1px Border.Input (N40). None: hanya border-bottom 2px saat focus/invalid. Invalid: Border.Danger (R300).

Glow biru lembut saat :focus — box-shadow: 0 0 0 2px var(--color-blue-b200). Tidak muncul pada appearance=none.

Micro Interaction

Transisi state memakai var(--transition-fast) (150ms). Coba interaksi langsung pada demo di bawah — hover, focus, ketik, dan tab antar field.

Hover
Border N40 → N100. Background tetap Background.Input.
Focus / Typing
Border B300 + ring 2px B200. Cursor biru saat mengetik.
Invalid
Border langsung ke Border.Danger (R300). Ring merah muda saat fokus.
Disabled
Background N20, teks & placeholder Text.Disabled (N50). Tidak ada hover/focus.

Informasi Penggunaan

Kapan dipakai
Untuk input teks pendek satu baris: nama, email, kode, pencarian inline. Gunakan Phone Number Field khusus untuk nomor HP +62. Gunakan textarea terpisah untuk teks panjang multi-baris.
Appearance & ukuran
standard untuk form utama. subtle untuk UI padat. none untuk inline edit. compact untuk toolbar/filter. monospaced untuk kode/token.
Aksesibilitas
Selalu sertakan <label> terlihat atau aria-label. Invalid: aria-invalid="true" + pesan error di bawah field. Jangan gunakan placeholder sebagai satu-satunya label.

Do & Don't

Panduan penggunaan text field yang benar untuk menjaga konsistensi formulir dan aksesibilitas.

Do — Lakukan ini
  • Gunakan label yang jelas di atas text field untuk memberikan konteks permanen.
  • Gunakan placeholder untuk contoh format input (mis. nama@email.com).
  • Berikan pesan error deskriptif di bawah field saat state invalid.
  • Pilih appearance=standard untuk formulir kompleks agar area input mudah ditemukan.
  • Gunakan token semantic — Border.Input, Border.Focused, Background.Disabled.
Don't — Hindari ini
  • Jangan menggunakan placeholder sebagai pengganti label — placeholder hilang saat mengetik.
  • Jangan gunakan appearance=none pada formulir kompleks tanpa petunjuk visual lain.
  • Jangan hardcode warna hex — gunakan CSS variable dari tokens.css.
  • Jangan menampilkan error hanya dengan warna border tanpa teks penjelasan.
  • Jangan membuat input custom dari div — gunakan <input> native.
