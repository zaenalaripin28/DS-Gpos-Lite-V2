Spacing System

Micro

3 tokens MICRO

Token terkecil dalam skala spacing. Digunakan untuk detail halus seperti border offset, nudge antar icon, dan hairline separator. Jangan gunakan untuk padding container.

Space.025
2px --space-025
Space.050
4px --space-050
Space.075
6px --space-075
📌 Kapan Dipakai
  • Focus ring offset & outline
  • Gap antar icon kecil (<16px)
  • Hairline separator / divider
  • Nudge posisi inline element
✅ Best Practice
  • 025 hanya untuk border / outline offset
  • 050 untuk gap icon dalam badge atau chip
  • 075 untuk subtle vertical nudge
✖ Hindari
  • Jangan pakai Micro untuk padding container
  • Jangan untuk section / page spacing
  • Hindari sebagai margin utama komponen

Small

2 tokens SMALL

Spacing untuk elemen kompak seperti badge, tag, chip, dan icon button. Memberikan padding yang tight tanpa mengorbankan keterbacaan.

Space.100
8px --space-100
Space.150
12px --space-150
📌 Kapan Dipakai
  • Padding badge, tag & chip
  • Padding button ukuran small
  • Gap antar item dalam list kompak
  • Padding input inline
✅ Best Practice
  • 100 (8px) untuk padding horizontal badge
  • 150 (12px) untuk padding vertical button small
  • Kombinasikan 100+150 untuk padding asimetris
✖ Hindari
  • Jangan untuk section spacing
  • Jangan untuk page margin / outer layout
  • Hindari sebagai gap antar card besar

Base

3 tokens BASE ★

Range default untuk sebagian besar kebutuhan UI — padding card, form spacing, dan gap antar elemen konten. Space.200 (16px) adalah nilai default yang direkomendasikan.

Space.200 ★
16px --space-200
Space.250
20px --space-250
Space.300
24px --space-300
📌 Kapan Dipakai
  • Padding card & container utama
  • Padding button default (medium/large)
  • Gap antar form field
  • Spacing antara elemen konten
✅ Best Practice
  • 200 (16px) sebagai padding default
  • 300 (24px) untuk padding card internal
  • Gunakan secara konsisten untuk semua form
✖ Hindari
  • Jangan campur Base dengan Micro dalam satu komponen
  • Jangan gunakan 300 sebagai page margin pada mobile
  • Hindari Base untuk section-level spacing

Large

2 tokens LARGE

Digunakan untuk memisahkan blok konten, section header, dan modal padding. Memberikan breathing room yang cukup antar area utama halaman.

Space.400
32px --space-400
Space.500
40px --space-500
📌 Kapan Dipakai
  • Gap antar section konten
  • Padding modal / dialog
  • Spacing antara content block
  • Margin bawah heading utama
✅ Best Practice
  • 400 (32px) untuk gap antar section kecil
  • 500 (40px) untuk padding modal & panel
  • Gunakan konsisten untuk semua section separator
✖ Hindari
  • Jangan gunakan di dalam komponen kecil
  • Jangan untuk padding teks inline
  • Hindari di konteks mobile yang ruang terbatas

XLarge

3 tokens XLARGE

Token untuk layout-level spacing: page margin, hero section padding, dan jarak antar blok utama. Digunakan pada level halaman, bukan di dalam komponen.

Space.600
48px --space-600
Space.800
64px --space-800
Space.1000
80px --space-1000
📌 Kapan Dipakai
  • Page margin & outer container
  • Hero section padding
  • Jarak antar major page section
  • Footer top padding
✅ Best Practice
  • 600 untuk page-level section spacing
  • 800–1000 hanya di layout / container level
  • Responsive: turunkan ke Large di layar mobile
✖ Hindari
  • Jangan gunakan di dalam komponen apapun
  • Jangan untuk gap antar row dalam list
  • Hindari di konteks compact / dense UI
