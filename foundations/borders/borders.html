Borders & Border Radius

8 corner radius token dari Corner.0 (0px) hingga Corner.Full (9999px) — beserta Border Width 2px style Inside untuk konsistensi visual di seluruh komponen GPOS Lite.

None

1 token NONE

Tanpa rounding — sudut persegi penuh. Digunakan untuk layout container, tabel, divider, dan komponen yang membutuhkan kesan tegas dan struktural.

Corner.0
0px --radius-0 rounded-none
📌 Kapan Dipakai
  • Tabel & data grid row
  • Divider & separator horizontal
  • Layout container full-bleed
  • Komponen edge-to-edge
✅ Best Practice
  • Gunakan konsisten untuk semua tabel row
  • Cocok untuk section background full-width
  • Pasangkan dengan border divider untuk struktur
✖ Hindari
  • Jangan untuk button & interactive element
  • Jangan sebagai card dalam layout modern
  • Hindari untuk modal & overlay component

Small

3 tokens SMALL

Radius terkecil untuk komponen kompak — memberikan subtle softening tanpa terlihat terlalu rounded. Cocok untuk elemen inline seperti badge, tag, dan checkbox.

Corner.025
2px --radius-025 rounded-sm
Corner.050
4px --radius-050 rounded
Corner.075
6px --radius-075 rounded-md
📌 Kapan Dipakai
  • Badge & status tag teks
  • Checkbox & radio button
  • Icon button ukuran kecil
  • Tooltip & inline label
✅ Best Practice
  • 025 (2px) untuk checkbox & form element kecil
  • 050 (4px) untuk badge & status chip
  • 075 (6px) untuk icon button & tooltip
✖ Hindari
  • Jangan untuk card besar atau modal
  • Jangan untuk primary button utama
  • Hindari pada hero section element

Medium

2 tokens MEDIUM ★

Range default untuk sebagian besar komponen UI — buttons, input fields, dan content card. Corner.100 (8px) adalah nilai default yang direkomendasikan untuk semua button dan form.

Corner.100 ★
8px --radius-100 rounded-lg
Corner.150
12px --radius-150 rounded-xl
📌 Kapan Dipakai
  • Button primary & secondary
  • Input field & textarea
  • Card konten & informasi
  • Dropdown & select menu
✅ Best Practice
  • 100 (8px) sebagai default untuk semua button
  • 150 (12px) untuk card & content container
  • Gunakan secara konsisten di seluruh form
✖ Hindari
  • Jangan campur 100 dan 150 dalam satu komponen
  • Jangan gunakan 150 untuk icon button kecil
  • Hindari untuk elemen yang butuh radius full

Large

1 token LARGE

Untuk komponen prominent seperti modal, dialog, dan bottom sheet. Memberikan kesan yang friendly, approachable, dan modern pada overlay element.

Corner.200
16px --radius-200 rounded-2xl
📌 Kapan Dipakai
  • Modal & dialog box
  • Bottom sheet & drawer
  • Side panel & flyout
  • Popover rich content
✅ Best Practice
  • Gunakan konsisten untuk semua overlay & modal
  • Pasangkan dengan backdrop untuk efek modern
  • Terapkan hanya pada corner yang terekspos
✖ Hindari
  • Jangan untuk button atau elemen inline
  • Jangan untuk komponen terlalu kecil (<48px)
  • Hindari pada elemen nested dalam komponen

Pill

1 token PILL

Radius penuh (9999px) untuk pill-shaped element. Memberikan kesan yang friendly dan modern. Digunakan untuk badge, status indicator, dan avatar circle.

Corner.Full
9999px --radius-full rounded-full
📌 Kapan Dipakai
  • Badge & status chip
  • Pill button & toggle switch
  • Avatar & user image circle
  • Progress bar & step indicator
✅ Best Practice
  • Selalu gunakan untuk semua badge status
  • Konsisten untuk semua avatar (circle shape)
  • Gunakan dengan padding proporsional
✖ Hindari
  • Jangan untuk card atau container besar
  • Jangan untuk modal & dialog
  • Hindari untuk input field utama

Border Width

1 style 2px INSIDE

GPOS Lite hanya menggunakan Border Width 2px dengan style Inside — border digambar di dalam bounds element sehingga tidak menambah ukuran atau mengganggu layout. Diimplementasikan dengan box-shadow: inset.

Tanpa Border

Element
background: #F5F6F7;
border-radius: 8px;

✓ 2px Inside Border

Element
box-shadow: inset 0 0 0 2px #1E7FD6;
border-radius: 8px;
Width 2px
Style Inside — tidak mengubah ukuran element (inset)
CSS box-shadow: inset 0 0 0 2px var(--color-border);
Tailwind ring-2 ring-inset ring-[#DFE2E6]
📌 Kapan Dipakai
  • Input field state (default, focus, error)
  • Button outlined & ghost variant
  • Card border & selection state
  • Focus ring indicator
✅ Best Practice
  • Selalu gunakan inset — tidak mengganggu layout
  • Kombinasikan dengan border-radius yang sesuai
  • Ganti warna sesuai state: default / focus / error
✖ Hindari
  • Jangan pakai border: 2px solid — menambah ukuran
  • Jangan campur dengan box-shadow dekorasi
  • Hindari border width lain (1px, 4px) di komponen
